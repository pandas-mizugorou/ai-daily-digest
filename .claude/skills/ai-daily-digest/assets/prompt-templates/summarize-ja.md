# 日本語要約プロンプト

選定された Top 5-7 件にのみ適用。コスト削減のため**選定後**にのみ呼ぶ。

## プロンプト本文

```
以下の英語記事情報から、ダイジェストサイト掲載用の日本語要約を作成してください。

【入力】
- title: <記事タイトル>
- source: <ソース名>
- summary_en: <英語要約>
- raw_excerpt: <記事本文の冒頭 1500 字>
- url: <URL>

【出力 JSON】
{
  "title_ja": "<記事タイトルの日本語訳。固有名詞・製品名はカタカナ化せず英語のまま (例: Claude, GPT-4, Llama 3)>",
  "summary_ja": "<3-5 行の日本語要約。「である調」。技術的事実を中心に、感想・推測・修飾語を排除>",
  "key_points_ja": [
    "<具体的な数字・名前を含む箇条書き 1>",
    "<具体的な数字・名前を含む箇条書き 2>",
    "<具体的な数字・名前を含む箇条書き 3 (最大 4 個)>"
  ],
  "tags": ["<記事のタグ 2-4 個 (英小文字、ハイフン区切り)>"]
}

【ルール】
- "summary_ja" は読み手が「3 行で要旨が掴める」ことを最優先
- "key_points_ja" は **具体的な数字・固有名詞** を必ず含める ("性能向上" ではなく "ベンチマーク X で 12% 向上")
- 推測・主観・"〜と期待される" のような修飾は避ける
- 元記事に書いていない情報は補わない
- 出力は JSON のみ。前置きや末尾の説明は不要
```

## モデル選定

要約は短く高品質に出したいので、Claude Sonnet を推奨（routine 環境では自動的に親モデルを使用）。

## エラー時のフォールバック

- summary_en だけで raw_excerpt が取れていない場合: summary_en をそのまま日本語訳
- 元記事が日本語ソース（Qiita / Zenn）の場合: title_ja = title、summary_ja は元記事冒頭から抜粋
- 完全に失敗した場合: 当該記事を選定から除外（push しない）
